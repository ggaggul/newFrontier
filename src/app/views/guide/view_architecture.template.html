본 문서는 Frontier의 뷰레이어 아키텍처에 대한 가이드로, 분석/설계자, 개발자, 아키텍트는 View Layer 개발시 본 가이드를 참조하여 개발표준을 준수한다
서비스 뷰레이어
  • dist : 운영 소스 배포 디렉토리
  • node_modules : 뷰레이어 빌더 라이브러리 디렉토리
  • src : 홈디렉토리
    • app : app root 디렉토리
      • components : app에서 공통으로 사용하는 open source ui component 및 navigation 정의 디렉토리
        • charts : chart component 디렉토리
        • common : navigation(sidebar, topnavi, footer) 디렉토리
        • datepicket : datepicker component 디렉토리
        • grid : grid component 디렉토리
        • iboxtools : iboxtools component 디렉토리
        • icheck : checkbox component 디렉토리
        • map : map component 디렉토리
      • services : angular service 디렉토리
        • templates : template 메뉴 관련 서비스 디렉토리
      • shared : angular ui component share module 디렉토리
      • views : 메뉴 화면 디렉토리
        • dashboard : dashboard template 디렉토리
        • forms : common form ui 렉토리
        • guide : frontier guide 디렉토리
        • tables : table & grid ui 디렉토리
        • template : CRUD Sample 디렉토리
      • app.component.ts : app main component 파일
      • app.helpers.ts : app helper(유틸성) 파일
      • app.module.ts : app main module 파일
      • app.routes.ts : app route(navigation) 파일
      • app.temaplte.html : app main template 파일
    • assets : 뷰레이어 Asset 디렉토리(CSS, Images, Font 등을 포함)
      • styles : Cascading Style Sheet 디렉토리
        • import : scss 파일 디렉토리
        • patterns : framework 기본 레이아웃 구성을 위한 이미지 디렉토리
      • images : Image 디렉토리
    • frontier.ts : frontier 어플리케이션 최상위 모듈 스크립트 파일
    • vendor.ts : frontier에서 사용되는 공용 오픈소스 프레임워크 및 라이브러리 선언 파일
  • vendor : npm으로 관리되지 않은 오픈소스 저장 디렉토리
  • index.html : frontier 어플리케이션 개발모드 welcome 파일
  • package.json : 뷰레이어 빌더 리소스 관리 파일
  • tsconfig.json : typescript 설정 파일
  • tslint.json : javascript 정적 테스트 설정 파일
  • webpack.config.json : frontier 어플리케이션 번들 설정 파일
