본 가이드는 코드의 품질, 생산성 향상 및 향후 유지보수를 위해 HTM의 포맷과 코딩 스타일에 대해서 정의한다.
본 가이드는 구글 코드의 스타일 가이드를 참고하여 작성한다.
일반적인 스타일 규칙
리소스

HTML을 통해 삽입(링크)되는 리소스는 프로토콜을 제한하며, 웹서버 상의 논리적인 절대경로를 사용한다.

 <script src="http://www.google.com/js/autotrack.js"></script>
 <script src="/js/autotrack.js"></script>
명명규칙

명명규칙은 다음을 따른다.

  • 파일명은 소문자와 숫자를 이용하여 작성하며 확장자(html)를 가지고, 의미 구분이 필요한 경우 밑줄(_)을 사용한다.
  • 디렉토리명은 소문자를 이용하여 작성하며, 의미 구분이 필요한 경우 밑줄(_)을 사용한다.
일반적인 포맷 규칙
들여쓰기

들어쓰기 기준은 4 spaces 이며, 탭과 혼용하여 사용하지 않는다.(1 tab은 4 spaces로 자동 변환한다.)

대소문자

모든 코드는 소문자로 작성한다. 단, text/CDATA와 대소문자가 유의미한 문자열은 제외한다.(HTML[태그, 속성, 속성값])

 <A HREF="/">Home</A>
 <img src="google.png" alt="Google">
불필요 공백 지우기

라인 뒤에 붙는 불필요한 공백은 모두 제거한다.

 <p>What?         </p>
 <p>Yes please</p>
HTML 스타일 규칙
문서타입(DocType)

XHTML을 대신, HTML5 문서타입을 사용한다.(내용이 없는 엘리먼트를 닫는 코드는 작성하지 않는다.)

 <!DOCTYPE html PUBLIC "-//W3C//DTD XHTML 1.0 Transitional//EN" "http://www.w3.org/TR/xhtml1/DTD/xhtml1-transitional.dtd">
  <br/>
 <!DOCTYPE html>
  <br>
HTML 유효성 검증

코드의 품질을 보장하기 위해 W3C HTML validator를 통해 검증된 코드를 사용한다.

의미에 맞는 엘리먼트 사용(Semantic element)

효율적인 코드와 접근성, 재사용성을 위해서 HTML 엘리먼트가 가진 본연의 의미에 맞는 태그를 사용한다. (레이아웃을 위해 table 대신 div, span를 사용한다.)

 <div onclick="goToRecommendations();">All recommendations</div>
 <a href="recommendations/">All recommendations</a>
멀티미디어 대비

접근성을 위해서 멀티미디어(이미지, 비디오 등) 리소스에 대한 대안을 제공한다.

 <img src="spreadsheet.png">
 <img src="spreadsheet.png" alt="Spreadsheet screenshot.">
관심사의 분리

HTML은 동작이나 디자인 코드가 아닌, 컨텐츠 구조만을 작성한다.(디자인:CSS, 동작:JavaScript)

  <!DOCTYPE html>
    <title>HTML sucks</title>
    <link rel="stylesheet" href="base.css" media="screen">
    <link rel="stylesheet" href="grid.css" media="screen">
    <link rel="stylesheet" href="print.css" media="print">
    <h1 style="font-size: 1em;">HTML sucks</h1>
    <p>I’ve read about this on a few sites but now I’m sure:
      <u>HTML is stupid!!</u>
    </p>
    <center>I can’t believe there’s no way to control the styling of
    my website without doing everything all over again!</center>

  <!DOCTYPE html>
    <title>My first CSS-only redesign</title>
    <link rel="stylesheet" href="default.css">
    <h1>My first CSS-only redesign</h1>
    <p>I’ve read about this on a few sites but today I’m actually
    doing it: separating concerns and avoiding anything in the HTML of
    my website that is presentational.</p>
    <p>It’s awesome!</p>
HTML 엔티티 코드

HTML 엔티티코드는 사용하지 않는다. 단, &lt;, &amp;, &nbsp;와 같이 HTML안에서 특별한 의미를 가진 코드는 예외로 한다. ( HTML EntityCode)

 The currency symbol for the Euro is &ldquo;&euro;&rdquo;.
 The currency symbol for the Euro is “€”.
type 속성

HTML5에서 CSS와 JavaScript 사용시 기본으로 설정되는, type 속성은 생략한다. (CSS기본type: text/css, JavaScript기본type: text/javascript)

  <link rel="stylesheet" href="/css/maia.css" type="text/css">
  <script src="/js/gweb/analytics/autotrack.js" type="text/javascript"></script>
  <link rel="stylesheet" href="//www.google.com/css/maia.css">
  <script src="/js/gweb/analytics/autotrack.js"></script>
ID, Class, Name 속성
  • ID와 Name은 Back-end의 데이터 처리를 위해 카멜 표기법(Camel Casing Notation)을 따른다.
  • Class는 소문자와 숫자로 작성하며, 이름이 복수의 단어로 구성될 경우, 구분자로 하이픈(-)을 사용한다.
HTML 포맷 규칙
일반적 포맷

블럭 엘리먼트(block element), 리스트(li), 테이블(table)은 새로운 줄에 작성하며, 하위 엘리먼트는 들여쓰기를 한다. (인라인 엘리먼트(inline element)의 경우, 줄바꿈 없이 가독성있게 작성한다.)

  <blockquote>
      <p><em>Space</em>, the final frontier.</p>
  </blockquote>

  <ul>
      <li>Moe</li>
      <li>Larry</li>
      <li>Curly</li>
  </ul>

  <table>
      <thead>
          <tr>
              <th scope="col">Income</th>
              <th scope="col">Taxes</th>
          </tr>
      </thead>
      <tbody>
          <tr>
              <td>$ 5.00</td>
              <td>$ 4.50</td>
          </tr>
      </tbody>
  </table>
HTML 따옴표

속성값을 설정할 때, 쌍따옴표("")를 사용한다.

  <a class='maia-button maia-button-secondary'>Sign in</a>
  <a class="maia-button maia-button-secondary">Sign in</a>
일반적인 메타 규칙
인코딩

UTF-8(no BOM)을 사용한다.(HTML:<meta charset="utf-8">)

주석

업무적인 설명이 필요한 곳에 주석을 작성한다. 단, 사람이 이해할 수 있는 명명규칙을 권고하며 주석은 최소화 한다.
